O que é o Módulo console?

Imagine que você está desenvolvendo um projeto em Node.js e precisa entender o que está acontecendo no seu programa — seja para exibir mensagens, rastrear erros ou medir o tempo de execução. É aí que o módulo console entra em cena, como um fiel companheiro para desenvolvedores. Neste guia, vamos mergulhar fundo no console do Node.js, explorando cada método e construtor, com explicações detalhadas e exemplos práticos. Prepare-se para dominar essa ferramenta essencial!

O módulo console no Node.js é como o canivete suíço da depuração. Ele oferece uma coleção de métodos para exibir mensagens no terminal (ou em outros destinos, como arquivos) e é parecido com o console que você já conhece dos navegadores, mas adaptado ao ambiente do servidor. Disponível globalmente, você não precisa importar nada para começar a usá-lo — basta chamar console.log("Olá!") e pronto.

Mas há mais! O console também é uma classe que você pode personalizar, permitindo redirecionar saídas para lugares específicos. Vamos explorar isso começando pelos construtores.

Criando Seu Próprio Console: Os Construtores

new Console(stdout[, stderr][, ignoreErrors])

O que faz: Cria uma instância personalizada do Console, definindo onde as mensagens normais (stdout) e os erros (stderr) serão exibidos. O parâmetro ignoreErrors decide se erros de escrita são ignorados (padrão: true).

Por que usar? Perfeito para quando você quer logs em arquivos ou fluxos personalizados, em vez do terminal padrão.

Como funciona: Você passa fluxos de escrita (como arquivos ou o terminal) e usa a instância como o console global.


const fs = require("node:fs");
const { Console } = require("node:console");

const output = fs.createWriteStream("./app.log");
const errorOutput = fs.createWriteStream("./error.log");

const myConsole = new Console(output, errorOutput);

myConsole.log("Aplicação iniciada");
myConsole.error("Erro crítico!");

new Console(options)

O que faz: Uma versão mais detalhada do construtor, aceitando um objeto com opções como stdout, stderr, colorMode, e mais.

Por que usar? Dá controle total sobre formatação (cores, indentação) e comportamento.

Como funciona: Passe um objeto com as configurações desejadas e use a instância para logs personalizados.


const fs = require("node:fs");
const { Console } = require("node:console");

const output = fs.createWriteStream("./custom.log");
const logger = new Console({
  stdout: output,
  stderr: output,
  colorMode: false,
  groupIndentation: 4
});

logger.group("Configuração");
logger.log("Sistema pronto");
logger.groupEnd();

Os Métodos do console: Sua Caixa de Ferramentas

Agora que entendemos como criar um Console, vamos explorar os métodos disponíveis — tanto no console global quanto nas instâncias personalizadas. Cada um tem um propósito único, e vou mostrar como usá-los com exemplos reais.

console.assert(value[, ...message])

O que faz: Testa se value é verdadeiro. Se for falso, exibe uma mensagem de erro no stderr.

Por que usar? Ideal para verificar condições durante o desenvolvimento.


const idadeMinima = 18;
let idadeUsuario = 16;

console.assert(idadeUsuario >= idadeMinima, "Usuário muito jovem: %d anos", idadeUsuario);
console.assert(true, "Isso não aparece");

console.clear()

O que faz: Limpa o terminal, mas só funciona se stdout for um TTY (terminal interativo).

Por que usar? Para deixar o terminal limpo e organizado.


console.log("Texto antigo");
console.clear();
console.log("Texto novo");

console.count([label])

O que faz: Conta quantas vezes foi chamado com um rótulo específico (padrão: "default").

Por que usar? Útil para rastrear repetições, como iterações em loops.


for (let i = 0; i < 3; i++) {
  console.count("passo");
}

console.countReset([label])

O que faz: Zera o contador de um rótulo específico.

Por que usar? Para reiniciar a contagem e reutilizar o rótulo.


console.count("evento");
console.count("evento");
console.countReset("evento");
console.count("evento");

console.debug(data[, ...args])

O que faz: Um alias para console.log(), mas com uma vibe de depuração.

Por que usar? Para destacar mensagens de debug no código.


console.debug("Debugging valor: %d", 42);

console.dir(obj[, options])

O que faz: Exibe uma representação detalhada de um objeto, usando util.inspect().

Por que usar? Para inspecionar objetos complexos de forma legível.


const usuario = { nome: "Ana", detalhes: { idade: 25 } };
console.dir(usuario, { depth: 1, colors: true });

console.dirxml(...data)

O que faz: Passa os argumentos para console.log() — não formata como XML, apesar do nome.

Por que usar? Uso genérico, mas sem função especial no Node.js.


console.dirxml("Dado", 123);

console.error([data][, ...args])

O que faz: Escreve mensagens de erro no stderr.

Por que usar? Para reportar problemas ou falhas.


console.error("Erro detectado: %s", "falha no servidor");

console.group([...label])

O que faz: Aumenta a indentação das próximas linhas, com rótulos opcionais.

Por que usar? Organizar saídas em blocos hierárquicos.


console.group("Processo");
console.log("Etapa 1");
console.group("Subdetalhes");
console.log("Item A");
console.groupEnd();
console.groupEnd();

console.groupCollapsed()

O que faz: Igual a group(), mas no Node.js não colapsa (diferente de navegadores).

Por que usar? Compatibilidade com código de navegadores.


console.groupCollapsed("Seção");
console.log("Dentro");
console.groupEnd();

console.groupEnd()

O que faz: Reduz a indentação atual.

Por que usar? Finalizar um grupo indentado.

Exemplo: Veja console.group() acima.

console.info([data][, ...args])

O que faz: Alias para console.log(), com tom informativo.

Por que usar? Para mensagens informativas gerais.


console.info("Sistema online desde: %s", new Date());

console.log([data][, ...args])

O que faz: O clássico — escreve mensagens no stdout com formatação.

Por que usar? Exibir resultados ou logs gerais.


console.log("Nome: %s, Pontos: %d", "João", 100);

console.table(tabularData[, properties])

O que faz: Exibe dados em formato de tabela.

Por que usar? Visualizar dados estruturados de forma clara.


const equipe = [
  { nome: "Lucas", cargo: "Dev" },
  { nome: "Clara", cargo: "Designer" }
];
console.table(equipe);

console.time([label])

O que faz: Inicia um temporizador para medir o tempo.

Por que usar? Analisar desempenho de trechos de código.


console.time("calculo");
for (let i = 0; i < 1000000; i++) {}
console.timeEnd("calculo");

console.timeEnd([label])

O que faz: Para o temporizador e exibe o tempo decorrido.

Por que usar? Finalizar a medição iniciada com time.

Exemplo: Veja console.time() acima.

console.timeLog([label][, ...data])

O que faz: Mostra o tempo parcial de um temporizador ativo.

Por que usar? Monitorar progresso em operações longas.


console.time("espera");
setTimeout(() => {
  console.timeLog("espera", "Passou metade");
}, 1000);

console.trace([message][, ...args])

O que faz: Exibe o rastreamento da pilha (stack trace).

Por que usar? Depurar, mostrando o caminho do código.


function testar() {
  console.trace("Onde estou?");
}
testar();

console.warn([data][, ...args])

O que faz: Alias para console.error(), com tom de aviso.

Por que usar? Alertar sobre situações potencialmente problemáticas.


console.warn("Atenção: %s está quase cheio", "disco");

Métodos Apenas no Inspector: Depuração Avançada

Esses métodos só funcionam com o flag --inspect, integrando-se ao Chrome DevTools.

console.profile([label])

O que faz: Inicia um perfil de CPU para análise de desempenho.

Por que usar? Identificar gargalos no código.


console.profile("Loop Pesado");
for (let i = 0; i < 100000; i++) {}
console.profileEnd("Loop Pesado");

console.profileEnd([label])

O que faz: Para o perfil de CPU.

Por que usar? Finalizar a análise iniciada com profile.

Exemplo: Veja console.profile() acima.

console.timeStamp([label])

O que faz: Adiciona um marco ao Timeline do inspector.

Por que usar? Marcar eventos para análise temporal.


console.timeStamp("Início da tarefa");

Conclusão

O módulo console do Node.js é mais do que apenas um log básico — é uma caixa de ferramentas completa para depuração e logging. Desde exibir mensagens simples até criar logs personalizados em arquivos, medir desempenho ou rastrear erros, ele tem tudo o que você precisa.
